第一章  项目概述

1.1 产品定位

PolyTrend 是一款专注于 PolyMarket 预测市场资讯聚合的微信小程序。核心价值是「信息降噪」——从海量的预测盘口中，自动筛选出最值得关注的事件，让用户用最少的时间掌握最新动态。

1.2 目标用户

  • PolyMarket 用户：希望快速了解市场热点与动向，辅助投注决策
  • 加密/预测市场爱好者：对全球重大事件预测结果感兴趣的人群
  • 资讯关注者：希望通过市场化方式了解时事热点概率的普通用户

1.3 核心功能概览

功能模块底部入口核心价值
🔥 热点Trending Tab24小时内最热门预测盘口 Top 10
⚡ 突发Breaking Tab24小时内波动最大的盘口 Top 10
🆕 最新New Tab24小时内最新上线的盘口 Top 10
🐳 巨鲸Whales Tab24小时内金额最大的 10 笔交易
👤 个人中心Profile Tab微信登录、收藏管理、语言设置

第三章  数据接口规范

3.1 PolyMarket 官方 API 基址

主数据 API
https://gamma-api.polymarket.com
订单数据

3.2 热点（Trending）接口

接口
GET /events?order=volume24hr&ascending=false&limit=10&active=true
字段名类型说明
idstring事件唯一 ID
titlestring事件标题
imagestring🌟 事件封面图 URL（用于卡片高级感展示）
volume24hrnumber过去24小时成交量（USD）
markets[].outcomePricesarray各选项当前投注比例，如 [0.72, 0.28]
markets[].outcomesarray各选项名称，如 ['Yes', 'No']
endDatestring盘口截止时间（ISO 格式）
urlstringPolyMarket 原文链接

3.3 突发（Breaking）接口

接口
GET /events?order=volume24hr&ascending=false&limit=10&active=true&tag=breaking

突发模块需额外展示波动趋势，关注以下字段：

字段名类型说明
markets[].priceHistoryarray价格历史时序数据（用于绘制迷你趋势图）
markets[].pricenumber当前价格（0–1 之间的概率值）
markets[].oneDayPriceChangenumber过去24小时价格变化量（正/负数）

3.4 最新（New）接口

接口
GET /events?order=startDate&ascending=false&limit=10&active=true
字段名类型说明
startDatestring盘口创建/上线时间（ISO 格式）
markets[].volumenumber盘口总成交量
markets[].liquiditynumber当前流动性（USD）

3.5 巨鲸（Whale Trades）接口

接口
GET https://data-api.polymarket.com/trades?limit=10&order=size&ascending=false&taker_only=true
字段名类型说明
idstring交易唯一 ID
sidestring🟢 BUY / 🔴 SELL（交易方向，卡片重点展示）
sizenumber交易金额（USD，用于排序）
pricenumber成交价格（0–1）
assetstring关联市场 ID（可反查事件名称）
timestampstring交易时间戳
transactionHashstring链上交易哈希（可点击跳转区块浏览器）

3.6 云函数核心代码示例

// 云函数：getTrending/index.js
const cloud = require('wx-server-sdk');
const axios = require('axios');
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV });

const API = 'https://gamma-api.polymarket.com/events'
  + '?order=volume24hr&ascending=false&limit=10&active=true';

exports.main = async (event, context) => {
  const db = cloud.database();

  // 检查5分钟缓存
  const cache = await db.collection('cache')
    .doc('trending').get().catch(() => null);
  if (cache && Date.now() - cache.data.ts < 300000) {
    return { code: 0, data: cache.data.list };
  }

  // 调用 PolyMarket API
  const res = await axios.get(API);
  const list = res.data.slice(0, 10);

  // 写入缓存
  await db.collection('cache').doc('trending')
    .set({ data: { list, ts: Date.now() } });

  return { code: 0, data: list };
};

image 字段使用规范

  • API 返回的 image 字段为 IPFS 或 CDN 链接，直接用 <image> 标签展示，比例固定 16:9，高度约 180rpx
  • image 为空时，用渐变色占位图（主色 → 辅助色渐变背景 + 事件首字母）替代，保证卡片高度统一
  • 添加 lazy-load 属性开启懒加载，避免一次性加载过多图片

热点模块 — 投注比例展示

  • 参考官网设计：进度条样式，Yes 端绿色，No 端红色
  • 文字标注：Yes 72% | No 28%
  • 多选盘口（>2个选项）：展示前3名 + 「其他」

突发/最新模块 — 趋势图展示

  • 使用 wx-charts 绘制微型折线图，高度约 80rpx
  • 仅展示最近 24 小时价格走势，涨为绿色，跌为红色
  • 右侧标注变化量：▲ +12.3% 或 ▼ -8.5%

巨鲸模块 — 交易卡片

  • 顶部大字显示交易金额：$250,000
  • 操作方向标签：BUY 或 SELL
  • 中部：关联事件名称 + 投注选项
  • 底部：交易时间 + 链上 Hash（点击跳转区块浏览器）

第五章  用户与账号模块

5.1 微信登录流程

1
用户进入小程序，未登录时展示「授权登录」引导页
2
用户点击「微信一键登录」，调用 wx.login() 获取临时 code
3
将 code 发送至云函数 getUserInfo
4
云函数调用微信 API，用 code 换取用户 openid 和 session_key
5
以 openid 为主键，在云数据库 users 集合中创建或更新用户记录
6
登录成功，跳转主页，用户头像/昵称展示在个人中心

第六章  国际化（多语言）方案

6.1 实现思路

维护全局语言配置文件，切换语言时更新 app.globalData.lang，所有页面从该变量读取对应语言文本，同时持久化到本地存储。

6.3 切换逻辑

// app.js
App({
  globalData: { lang: 'zh-CN', i18n: {} },
  onLaunch() {
    const saved = wx.getStorageSync('lang') || 'zh-CN';
    this.switchLang(saved);
  },
  switchLang(lang) {
    this.globalData.lang = lang;
    this.globalData.i18n = require(`./i18n/${lang}.js`);
    wx.setStorageSync('lang', lang);
    // 通知所有页面刷新
    getCurrentPages().forEach(p => p.onLoad && p.onLoad(p.options));
  }
});

8.2 微信小程序上线步骤

  1. 在 mp.weixin.qq.com 注册小程序，类目选择「工具 > 效率/信息聚合」
  2. 完成主体认证（个人：身份证实名；企业：营业执照）
  3. 使用微信开发者工具开发并完成本地测试
  4. 在「开发设置 > 服务器域名」配置白名单（或使用云函数代理绕过）
  5. 提交代码至体验版，邀请测试人员体验并修复问题
  6. 提交审核，填写功能介绍（强调「资讯聚合」属性）
  7. 审核通过后（通常 1–7 个工作日）点击发布上线

8.3 重要注意事项

⚠️ 域名白名单
小程序只能请求已配置域名。推荐使用云函数作为 API 代理，可完全绕过白名单限制，同时保护 API 调用逻辑。
⚠️ 审核合规
产品定位为「资讯聚合工具」，审核材料中避免出现「投注」「押注」「下注」等敏感词，突出信息展示功能。
💡 性能优化
定时云函数每5分钟主动刷新缓存，保证用户打开时数据已就绪，避免感知到加载等待时间。

第九章  常见问题 Q&A

问题解答
PolyMarket API 需要 Key 吗？ 大部分公开接口无需 API Key，直接 GET 请求即可。建议在云函数中加入请求节流（每5分钟一次），避免被限流或 IP 封禁。
图片加载慢怎么办？ 在 <image> 标签添加 lazy-load 属性开启懒加载；封面图可通过微信云存储中转缓存至国内 CDN 加速。
数据实时性如何保证？ 定时云函数每5分钟更新缓存；用户下拉刷新时可强制绕过缓存直接请求 API，保证最新数据。
多语言切换会影响性能吗？ 不会。语言包文件极小（< 5KB），App 启动时一次性加载进内存，切换时仅更新变量并触发页面数据刷新，无重载成本。
用户数据如何保证安全？ 微信云数据库默认对每条记录设置 openid 权限隔离，用户只能读写自己的数据。本产品不存储任何敏感金融信息。
WebView 跳转需要配置什么？ 需在小程序后台「开发设置 > 业务域名」中将 polymarket.com 添加为白名单，并完成域名校验文件配置。
我是小白，从哪里开始学习？ 推荐路径：① 阅读微信官方文档 developers.weixin.qq.com → ② 注册账号开通云开发 → ③ 按本文档Phase 1~5逐步推进 → ④ 遇到问题搜索「微信小程序 + 关键词」。
